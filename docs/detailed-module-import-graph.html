mlf4 / mlf2 Detailed Module Import Graph

这是更细的 module-to-module 版本：不再只画 domain，而是直接把模块节点展开成一个聚焦子图。为了可读性，我没有硬塞进 repo 全部 190+ 个 Haskell 模块；这张图聚焦公开入口、它们的近邻、以及高连接度核心模块，能更真实地看出具体 import 链条。

How this subset was picked

  • • Starts from `MLF.API`, `MLF.Pipeline`, `MLF.Program`, `MLF.XMLF`, and `Main`.
  • • Adds direct imports, direct consumers, plus one extra hop from the public modules.
  • • Also adds top-degree implementation hubs so important internals stay visible.
  • • Result: 61 modules / 224 edges — much finer than the domain diagram, but still viewable.

What pops out

  • • `MLF.Frontend.Syntax` has very high fan-in in this slice.
  • • `MLF.Elab.Pipeline` and `MLF.Elab.Run.Pipeline` are strong orchestration hubs.
  • • `MLF.Constraint.Types` / `MLF.Constraint.Presolution` remain central semantic anchors.
  • • `MLF.Reify.TypeOps` is a key support dependency for many elaboration modules.

Reading tip

  • • Left → right roughly follows façade → frontend → constraints → elaboration → support.
  • • Arrow density is real import density, not conceptual arrows I invented.
  • • Each node shows `in/out` degree within this focused subgraph only.
  • • Use the tables below when the visual graph gets too dense.
