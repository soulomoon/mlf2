## Gate Rules

- A row is `YES` only when behavior is objectively aligned with the thesis, the active code/test anchors still match the claim, and the required verification passes.
- Use only `YES` or `NO` (never `PARTIAL`, `N/A`, or prose gates).
- Keep mechanism order fixed; improving-loop rounds always anchor on the first `NO` row.

## Verification Cadence

- Row-focused verification must use the thesis/code/test anchors named in the row.
- Safety/regression verification should prefer the narrowest relevant guard suite first.
- Any code-changing attempt must end with the full gate: `cabal build all && cabal test`.
